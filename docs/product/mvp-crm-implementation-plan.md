# План реализации MVP CRM для частных преподавателей иностранных языков

Этот документ описывает этапы реализации продукта, очередность работ и текущий прогресс.

Бизнес-требования, основная модель продукта и пользовательские сценарии описаны в [docs/product/mvp-crm-plan.md](/Users/viig/www/teachly/docs/product/mvp-crm-plan.md).

Для наглядного отслеживания прогресса каждый этап и подэтап отмечается одним из двух состояний:

### [x] Этап 4. Технический фундамент

Цель этапа:

- подготовить основу для реализации первой версии продукта
- запустить минимальный каркас системы

На этом этапе в работу идут только те части системы, которые нужны немедленно:

- авторизация (JWT, используем passport, @nestjs/passport, local strategy, jwt strategy)
- пользователь преподавателя
- базовая основа frontend и backend

Принцип этапа:

- не создавать домены заранее "про запас"
- добавлять новый домен только тогда, когда он нужен следующему продуктовому этапу

Подэтапы:

- [x] подготовить базовую структуру backend
- [x] подготовить базовую структуру frontend
- [x] подготовить окружение для поэтапной разработки
